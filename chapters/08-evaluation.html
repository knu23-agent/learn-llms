📊 Evaluating LLMs

"How good is this model?" turns out to be one of the hardest questions in AI. LLMs can write poetry, debug code, and reason about ethics — so how do you measure that? This chapter covers the full toolkit: automated metrics, benchmarks, LLM-as-judge, and human eval.

Why Evaluation is Hard

The core problem: For most generation tasks, there's no single correct answer. "Write a poem about autumn" has infinite valid responses. Unlike classification (right/wrong), LLM quality lives on a spectrum — and humans don't always agree either.

The three main evaluation paradigms:

  1. Automated metrics — fast, cheap, reproducible. But often miss what matters.
  2. Benchmark tasks — standardized tests with known answers. Gameable.
  3. Human / LLM-as-judge evaluation — more valid, but expensive and slow.

Automated Metrics

Perplexity

Perplexity measures how "surprised" a language model is by a test set. Lower is better — it means the model assigns high probability to the actual text.

# Perplexity = exp(average negative log-likelihood)
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

model = AutoModelForCausalLM.from_pretrained("gpt2")
tokenizer = AutoTokenizer.from_pretrained("gpt2")

text = "The cat sat on the mat."
inputs = tokenizer(text, return_tensors="pt")

with torch.no_grad():
    outputs = model(**inputs, labels=inputs["input_ids"])

loss = outputs.loss              # average NLL per token
perplexity = torch.exp(loss)
print(f"Perplexity: {perplexity.item():.2f}")

Limitation: Perplexity only tells you if the model assigns high probability to the test data — not whether it's helpful, accurate, or safe.

BLEU Score

Originally designed for machine translation. Compares n-gram overlap between the generated text and one or more reference translations.

from nltk.translate.bleu_score import sentence_bleu

reference = [["the", "cat", "is", "on", "the", "mat"]]
candidate = ["the", "cat", "sits", "on", "the", "mat"]

score = sentence_bleu(reference, candidate)
print(f"BLEU: {score:.3f}")  # 0.0 - 1.0 (higher is better)

Limitation: BLEU penalizes valid paraphrases. "The feline rests on the rug" scores near zero even if it's an excellent translation.

ROUGE

Recall-Oriented Understudy for Gisting Evaluation. Used for summarization. ROUGE-1 measures unigram overlap; ROUGE-L measures longest common subsequence.

from rouge_score import rouge_scorer

scorer = rouge_scorer.RougeScorer(['rouge1', 'rougeL'], use_stemmer=True)
reference = "The quick brown fox jumps over the lazy dog."
generated = "A fast brown fox leaps over a sleepy dog."

scores = scorer.score(reference, generated)
print(scores['rouge1'])   # Score(precision, recall, fmeasure)

BERTScore

Uses BERT embeddings to compare semantic similarity between generated and reference text. Much better at capturing meaning than n-gram overlap.

from bert_score import score

candidates = ["The feline rests on the rug."]
references  = ["The cat is on the mat."]

P, R, F1 = score(candidates, references, lang="en")
print(f"BERTScore F1: {F1.mean():.4f}")  # Much higher than BLEU!

Standardized Benchmarks

Academic benchmarks give models known problems with known answers — enabling reproducible comparisons:

BenchmarkTestsFormat
MMLU 57 subjects from school to professional exams Multiple choice
HumanEval Python coding problems, unit-tested Code generation
GSM8K Grade-school math word problems Chain-of-thought reasoning
HellaSwag Common-sense sentence completion Multiple choice
TruthfulQA Questions humans often answer incorrectly True/false + generation
MT-Bench Multi-turn conversations across 8 categories Open-ended, LLM-judged
LMSYS Chatbot Arena Head-to-head model battles, human voting Elo rating
Benchmark contamination: If training data contains benchmark answers, scores are inflated. Always check whether a model was trained after the benchmark was published. The leaderboard problem: models can be over-optimized for specific benchmarks while remaining poor in real-world use.

LLM-as-Judge

Use a strong LLM (like GPT-4 or Claude) to evaluate another model's outputs. Much cheaper than human evaluation, surprisingly correlated with human judgment.

from openai import OpenAI

client = OpenAI()

def llm_judge(question, answer, criteria="helpfulness, accuracy, clarity"):
    prompt = f"""You are an expert evaluator. Rate the following answer on a scale of 1-10.

Question: {question}

Answer: {answer}

Evaluate on: {criteria}

Respond with:
Score: [1-10]
Reasoning: [brief explanation]"""

    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": prompt}],
        temperature=0,
    )
    return response.choices[0].message.content

Known biases to watch for:

  • Position bias — judges favor the first response listed
  • Verbosity bias — longer answers score higher regardless of quality
  • Self-enhancement bias — a model may prefer its own style

Mitigate by: randomizing response order, averaging multiple judge runs, using diverse judge models.

Human Evaluation

The gold standard — but expensive. Key guidelines:

  • Define clear rubrics before collecting ratings
  • Use multiple annotators and measure inter-annotator agreement (Cohen's κ)
  • Blind evaluators to which model produced which output
  • Representative prompts — your eval set should reflect your actual use cases
  • Adversarial prompts — include edge cases and attempts to elicit bad behavior

Safety Evaluation

Alignment isn't free. Even well-trained models can produce harmful, biased, or factually incorrect outputs. Safety evaluation is now a mandatory part of responsible deployment.

Key safety dimensions to test:

  • Refusal rate — does the model refuse clearly harmful requests?
  • Hallucination rate — how often does it confidently state falsehoods?
  • Bias — does it treat demographic groups differently?
  • Jailbreak robustness — does it maintain safety under adversarial prompts?
  • Toxicity — does it produce offensive or harmful content unprompted?
# Simple hallucination detection: ask the model about a known fact,
# then verify the answer against a ground truth source

questions_with_answers = [
    ("What year was Python created?", "1991"),
    ("Who wrote Pride and Prejudice?", "Jane Austen"),
