💬 Language Models & LLMs

A language model is a system that assigns probabilities to text. From that simple idea — predict what comes next — emerges GPT-4, Claude, Gemini, and everything in between.

The Core Task: Next-Token Prediction

Key idea: Given everything you've read so far, what word comes next? A language model does exactly this — but with overwhelming scale.

That's literally it. The entire training objective is:

# Given tokens [t1, t2, t3, ...], predict t_next
P(t_next | t1, t2, t3, ...)

Train on enough text — trillions of tokens from the web, books, code — and something remarkable happens: the model doesn't just learn to predict words. It learns facts, reasoning patterns, languages, coding skills, and more.

A Brief History

N-gram models (pre-2010): Count how often word sequences appear. "The cat sat on the ___" → look up what word most commonly follows in your corpus. Fast, simple, but can't capture long-range context.

RNN language models (2013–2017): Process tokens one at a time, maintaining a hidden state. Better at long sequences, but still struggle with very long-range dependencies.

Transformer LLMs (2017–present): Attention over all tokens simultaneously. Scale up the architecture and data, and you get GPT, BERT, Claude, etc.

Tokenization

LLMs don't read raw text — they read tokens. A token is roughly a word or word-piece. The tokenizer converts text to a sequence of integer IDs.

# Example: GPT-2 tokenizer
from transformers import GPT2Tokenizer

tokenizer = GPT2Tokenizer.from_pretrained("gpt2")
text = "Hello, neural networks!"
tokens = tokenizer.encode(text)
# → [15496, 11, 17019, 7127, 0]
# → ["Hello", ",", " neural", " networks", "!"]

Why not just use words? Because tokens let the vocabulary handle:

  • Unknown words (split into sub-pieces: "unbelievable" → "un", "believ", "able")
  • Multiple languages in one vocabulary
  • Code with unusual identifiers

Context Window

Every LLM has a context window — the maximum number of tokens it can "see" at once. This is the model's working memory:

  • GPT-2: 1,024 tokens (~750 words)
  • GPT-3: 2,048 tokens
  • GPT-4: up to 128,000 tokens
  • Claude 3.5: up to 200,000 tokens (~150,000 words — a full novel)

Beyond the context window, the model has no memory. This is why long conversations sometimes require techniques like summarization or retrieval.

Scaling Laws

Kaplan et al. (2020): Model performance improves predictably with more parameters, more data, and more compute — following smooth power laws. Bigger models trained on more data almost always perform better.

This is why the field moved from millions to billions to trillions of parameters. Scale alone produced capabilities no one explicitly programmed: translation, reasoning, coding, math — all emerging from "just" predicting the next token.
