👁️ Attention & Transformers

The Transformer architecture — introduced in "Attention Is All You Need" (2017) — is the engine behind every modern LLM. Its secret weapon: the attention mechanism.

The Problem with RNNs

Before Transformers, the dominant approach was Recurrent Neural Networks (RNNs). They processed text one token at a time, passing a hidden state forward.

The problem: By the time an RNN reaches token 500, information from token 1 has been squashed into a tiny vector and mostly forgotten. Long-range dependencies — like resolving "it" in a sentence from 10 lines ago — were hard.

Attention: Looking at Everything at Once

Attention solves this by letting every token look directly at every other token, regardless of distance. No sequential processing — all tokens in parallel.

The Library Analogy

Imagine you're searching a library:

  • Query (Q) — your search query ("books about cats")
  • Keys (K) — the title/description of each book on the shelf
  • Values (V) — the actual content of each book

Attention computes how well your query matches each key, then returns a weighted blend of the values — more content from books that matched your query better.

The Attention Formula

Attention(Q, K, V) = softmax( Q·Kᵀ / √d_k ) · V

Breaking it down:

  1. Q·Kᵀ — dot product between queries and keys. High value = good match.
  2. / √d_k — scale down to prevent huge values from dominating softmax.
  3. softmax(…) — convert scores to probabilities (sum to 1). These are the "attention weights".
  4. · V — weighted sum of values. The output for each token.
# Minimal attention in NumPy
import numpy as np

def attention(Q, K, V):
    d_k = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(d_k)          # [seq, seq]
    weights = np.exp(scores) / np.exp(scores).sum(axis=-1, keepdims=True)
    return weights @ V                         # [seq, d_v]

Multi-Head Attention

Instead of one attention computation, Transformers run attention multiple times in parallel (e.g., 8 or 16 "heads"), each learning different relationship types:

  • One head might track syntactic relationships (subject → verb)
  • Another might track coreference (pronoun → noun)
  • Another might track positional patterns

All heads' outputs are concatenated and projected.

The Transformer Block

One Transformer layer = Multi-Head Self-Attention + Feed-Forward Network, with residual connections and layer normalization around each:

x = x + MultiHeadAttention(LayerNorm(x))
x = x + FeedForward(LayerNorm(x))

Stack 12–96 of these layers (GPT-3 has 96), and you have an LLM.
