📉 Training: How Networks Learn

A neural network starts with random weights and knows nothing. Training is the process of nudging those weights, step by step, until the network gets good at its job.

The Loss Function

Key idea: We need a way to measure how wrong the network is. That's the loss function — a single number that says "here's how badly you're doing."

Two common loss functions:

Mean Squared Error (MSE) — used for regression (predicting numbers):

# Average of (prediction - actual)² across all examples
def mse_loss(predictions, targets):
    return ((predictions - targets) ** 2).mean()

Cross-Entropy Loss — used for classification (predicting categories):

# Measures the "surprise" of predicting the wrong class
import numpy as np

def cross_entropy(probs, true_class):
    return -np.log(probs[true_class])

The lower the loss, the better the network is doing. Our goal: minimize the loss.

Gradient Descent — The Foggy Hillside

Analogy: Imagine you're lost in dense fog on a hilly landscape. You can't see the valley (minimum loss), but you can feel which direction is downhill. So you take a small step downhill. Then check again. Repeat. Eventually you reach the bottom.

That's gradient descent. The "hill" is the loss landscape. The gradient tells us which direction is "uphill" for each weight, so we step in the opposite direction.

The update rule for each weight w:

w = w - learning_rate × gradient

In code:

for param in model.parameters():
    param -= learning_rate * param.grad

The Learning Rate

The learning rate controls the step size. It's one of the most important hyperparameters:

  • Too large: You overshoot the minimum and bounce around (or diverge entirely)
  • Too small: Training works but takes forever
  • Just right: Steady, efficient descent toward a good solution

Backpropagation

Key idea: Backpropagation is just the chain rule from calculus applied backwards through the network. It efficiently computes "how much did each weight contribute to the error?"

The process:

  1. Forward pass — run the input through the network, get a prediction
  2. Compute loss — compare prediction to the true answer
  3. Backward pass — compute gradients from output layer back to input layer
  4. Update weights — subtract learning_rate × gradient from each weight
  5. Repeat — for thousands of examples across many epochs
# PyTorch training loop (the real thing)
for epoch in range(num_epochs):
    for inputs, targets in dataloader:
        optimizer.zero_grad()          # clear old gradients
        predictions = model(inputs)    # forward pass
        loss = criterion(predictions, targets)  # compute loss
        loss.backward()                # backpropagation
        optimizer.step()               # update weights
